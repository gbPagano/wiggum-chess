## Codebase Patterns
- FEN parsing: rank 7 (black's back rank) is the first rank in FEN notation; iterate rank from 7 down to 0
- En passant encoding: `en_passant_file` stores just the file index; the target rank is determined by whose turn it is (White to move: target rank 5; Black to move: target rank 2)
- Castling detection in apply_move: king moves exactly 2 files → castling; pawn diagonal capture to empty square → en passant
- Perft reference values: always verify against chesslib using `cargo test -p chesslib get_depth4_perft_refs -- --ignored --nocapture` rather than hardcoding guesses
- `cargo test --workspace -- --skip gen_files::magics::name` is the correct full workspace test command; analysis spawn test may fail in environments without compiled engine binaries (pre-existing, not introduced by this work)
