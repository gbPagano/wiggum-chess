## Codebase Patterns
- Cargo is not pre-installed in devcontainer — install with `curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y`, then `source "$HOME/.cargo/env"`
- Workspace root `Cargo.toml` holds profiles; subcrate `Cargo.toml` files should NOT have `[profile.*]` sections
- The test `gen_files::magics::name` intentionally has `assert!(false)` — it's a dev scratchpad, always fails, not a regression
- Run tests with `--skip gen_files::magics::name` to exclude it; all other tests should pass
- `cargo build --workspace`, `cargo test --workspace`, `cargo bench --bench perft -p chesslib` are the key quality checks
- Build-time code generation: add new tables by creating `gen_files/foo.rs`, exporting from `gen_files/mod.rs`, calling `write_foo(&mut f)?` in `build.rs`, and adding accessor functions in `magic.rs` (which includes the generated file via `include!(concat!(env!("OUT_DIR"), "/magic_file.rs"))`)
- BitBoard: use `get_squares()` iterator to enumerate set bits; no `Sub` trait — can't do `bb - BitBoard(1)` in safe code
- Zobrist keys are in `ZOBRIST_PIECES[piece*128 + color*64 + sq]`, `ZOBRIST_CASTLING[0..4]` (WK,WQ,BK,BQ), `ZOBRIST_EN_PASSANT[file]`, `ZOBRIST_SIDE`
