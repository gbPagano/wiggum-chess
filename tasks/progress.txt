## Codebase Patterns
- Evolution session ids should use a UTC timestamp in `YYYYMMDDTHHMMSSZ` format so artifact directories sort chronologically and stay deterministic within a run.
- Session-level artifacts live at the session root: keep human-readable summary state in `summary.md` and selected shell inputs in `session.env`.
- Iteration artifacts live under `iterations/<N>/` and should be created by the orchestration script before any skill runs so every phase can rely on stable file paths.
- For shell-story quality gates here, validate with `bash -n scripts/evolution-loop.sh`, a workspace Cargo build, and `cargo test --workspace -- --skip gen_files::magics::name`.
- Correctness-gate results belong under `iterations/<N>/correctness/results.md`, and `iteration.json` should mirror them with `correctness.passed` plus `correctness.benchmarkEligible` so later phases can skip benchmarking deterministically.
- Benchmark policy metadata should be explicit in `iteration.json`: record `benchmark.policyStage` and `benchmark.sufficientForPromotion` so the decision phase can distinguish screening evidence from promotion-ready confirmation.
